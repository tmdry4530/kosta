# PRD — When Whales Move

> 고래 자금 이동과 시장 이벤트의 상관관계를 보여주는 인터랙티브 대시보드
> 학교 수업 과제 (크롤링 + 데이터 분석) — 2026년 봄학기

---

## 1. 한 줄 요약

**"대형 지갑(고래)들은 시장이 흔들릴 때 무엇을 했을까?"** 라는 질문에 데이터로 답하는, 스토리텔링 중심의 웹 대시보드.

## 2. 만드는 이유 (Why)

- **수업 과제 요구사항 충족**: 크롤링 + 데이터 분석 + 시각화의 전 과정을 보여줘야 한다.
- **기존 블록체인 인덱서 경험을 활용**하되, 인덱서 직접 구축 대신 Dune Analytics의 인덱싱된 데이터를 사용해 시간을 절약한다.
- **포트폴리오 가치**: Web3에 한정되지 않은, 일반 데이터 분석/시각화 프로젝트로도 어필 가능.
- **차별점**: 단순 거래량 차트가 아니라 **역사적 사건과 고래 행동의 인과/패턴**을 엮는 내러티브.

## 3. 타깃 사용자

| 사용자 | 기대 |
|---|---|
| 블록체인 비전공 일반인 | "고래가 뭐고 이게 왜 흥미로운지" 가 입문자에게도 잡혀야 함 |
| 수업 평가자 (교수/조교) | 데이터 파이프라인의 완결성, 분석의 깊이, 시각화 품질 |
| 채용 담당자 | 기술 스택 폭(Python + Node + React), 도메인 깊이, 결과물 완성도 |

## 4. 성공 기준 (Definition of Done)

- ✅ 21개 이벤트(15 글로벌 + 6 한국) 각각에 대해 ±7일 윈도우 분석이 자동으로 렌더링된다.
- ✅ 사용자가 임의의 이벤트를 클릭하면 해당 시점의 고래 활동 변화가 차트로 표시된다.
- ✅ 거래소 입금/출금 방향 분석이 최소 1개 차트로 제공된다.
- ✅ 모든 데이터는 PostgreSQL을 통해 서빙되며, 페이지 로딩이 3초 이내.
- ✅ README에 따라 누구나 `docker compose up && pnpm install && pnpm dev` 수준으로 재현 가능.
- ✅ 블록체인을 모르는 동료가 5분 안에 "이 사이트가 뭘 보여주는지" 이해할 수 있다.

## 6. 핵심 사용자 시나리오 (User Stories)

### US-1. "FTX 파산 직전 고래들은 뭘 했나?"
- 사용자가 이벤트 타임라인에서 "FTX 파산" 카드를 클릭한다.
- 화면 중앙에 ±7일 윈도우의 고래 전송량, 거래소 입금 비중이 표시된다.
- 옆에는 "전 주 대비 거래소 입금 +N% 증가" 같은 요약 인사이트가 나온다.

### US-2. "전체 흐름을 한눈에"
- 메인 페이지에서 2017~2026 전체 기간의 주간 고래 전송량 차트가 보인다.
- 차트 위에 21개 이벤트가 점/세로선으로 오버레이된다.
- 마우스 오버 시 이벤트명과 해당 주의 수치가 툴팁으로 나온다.

### US-3. "패턴이 있나?"
- 사용자가 "이벤트 유형별 비교" 탭으로 이동한다.
- 폭락/호재/위기/과열 4개 카테고리별로 평균 고래 행동 패턴이 비교된다.

### US-4. "한국은 다른가?"
- "한국 섹터" 탭에서 한국 특화 이벤트 6개의 영향을 본다.
- 김치프리미엄 시기 등에서 글로벌 평균과의 차이를 확인한다.

## 7. 기능 요구사항 (Functional Requirements)

### FR-1. 데이터 파이프라인 (Python)
- **FR-1.1** Dune Analytics에서 주간 고래 전송 데이터를 받아 PostgreSQL에 적재한다.
- **FR-1.2** CoinGecko 또는 대체 소스에서 ETH/BTC 일별 가격을 받아 적재한다. (현재 401 이슈 해결 필요)
- **FR-1.3** 거래소 입출금 방향 데이터를 Dune 추가 쿼리로 받아 적재한다.
- **FR-1.4** `EVENTS.md` 카탈로그를 시드 데이터로 적재한다.
- **FR-1.5** 모든 적재 작업은 멱등(같은 입력 → 같은 결과). 재실행 안전.
- **FR-1.6** **크롤링은 Cloudflare Browser Rendering `/crawl` 엔드포인트로 수행한다.** 자체 스크레이퍼 작성 금지.
- **FR-1.7** 21개 이벤트 각각의 ±7일 창에 대해 글로벌(CoinDesk, Cointelegraph) + 한국(토큰포스트, 블록미디어) 뉴스를 크롤하여 `news_articles` 테이블에 적재한다.
- **FR-1.8** `alternative.me`에서 공포·탐욕 지수 히스토리를 크롤하여 `fear_greed_index` 테이블에 적재한다.

### FR-2. API (Node.js)
- **FR-2.1** `GET /api/events` — 이벤트 목록 (필터: 카테고리, 지역, 기간).
- **FR-2.2** `GET /api/events/:id/window` — 특정 이벤트 ±N일 윈도우의 고래/거래소 데이터.
- **FR-2.3** `GET /api/whale-flows?from=&to=&asset=` — 기간/자산별 고래 전송 시계열.
- **FR-2.4** `GET /api/cex-flows?from=&to=` — 거래소 입출금 시계열.
- **FR-2.5** `GET /api/prices?asset=&from=&to=` — 가격 시계열.
- **FR-2.6** `GET /api/events/:id/news?limit=` — 해당 이벤트 ±7일 창의 뉴스 헤드라인.
- **FR-2.7** `GET /api/fear-greed?from=&to=` — 공포·탐욕 지수 시계열.
- **FR-2.8** 모든 응답은 5분간 인메모리 캐시 (외부 API가 없으니 부하는 낮음).

### FR-3. 프론트엔드 (React)
- **FR-3.1** **메인 화면**: 전체 기간 고래 전송량 + 이벤트 오버레이.
- **FR-3.2** **이벤트 디테일**: 클릭 시 사이드 패널에 ±7일 분석 표시.
- **FR-3.3** **이벤트 비교 뷰**: 카테고리별 평균 패턴 비교.
- **FR-3.4** **한국 섹터 뷰**: 한국 이벤트 전용 페이지.
- **FR-3.5** 모바일 대응은 "최소한의 가독성"만 (데스크톱 우선).

## 8. 비기능 요구사항 (Non-Functional Requirements)

- **재현성**: 새로운 사람이 README만으로 30분 안에 로컬에서 띄울 수 있어야 함.
- **성능**: 페이지 첫 로딩 3초 이내, 이벤트 클릭 응답 500ms 이내.
- **타입 안전성**: TypeScript strict, Python mypy strict.
- **개발자 경험**: `pnpm dev` 한 번으로 web + api 동시 실행.

## 9. 범위 외 (Out of Scope)

이번 과제에서 **하지 않는다**:

- ❌ 실시간 데이터 (배치 적재만, 갱신은 수동 스크립트 실행).
- ❌ 사용자 인증 / 회원가입.
- ❌ 프로덕션 배포 (학교 발표용으로 로컬 또는 Vercel/Render 무료 티어 데모만).
- ❌ 모바일 네이티브 앱.
- ❌ 비트코인 체인 분석 (이더리움만).
- ❌ DeFi 프로토콜 내부 거래 분석 (단순 transfer만).
- ❌ AI/ML 기반 이상 탐지 (단순 통계 비교만).

## 11. 위험 요소 (Risks)

| 위험 | 영향 | 대응 |
|---|---|---|
| Dune 무료 크레딧 소진으로 추가 쿼리 불가 | 중 | 받은 CSV 우선 활용, 추가 쿼리는 꼭 필요할 때만 |
| 가격 데이터 소스(CoinGecko) API 제한 | 중 | 이미 받아둔 데이터 사용, 보강 필요 시 yfinance 백업 |
| 거래소 지갑 식별의 부정확성 | 중 | Dune 쿼리에서 라벨링된 주소만 사용한다고 명시 |
| 21개 이벤트의 "정확한 영향 시작 시점" 모호 | 저 | 이벤트마다 ±7일 윈도우 사용으로 통일 |
| 학기 일정상 시간 부족 | 고 | DoD를 보수적으로 잡고, "있으면 좋은 것"은 PLAN의 후반부로 |
