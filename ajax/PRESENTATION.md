# WeatherEats 발표 스크립트

## 1. 인트로 (약 30초)

네. 프로젝트 발표 시작하겠습니다.

---

WeatherEats는 사용자의 현재 위치를 분석해 현재 날씨에 어울리는 맛집을 추천하고, 해당 메뉴를 기준으로 주변 맛집을 탐색할 수 있는 웹 서비스입니다.


## 2. 기획 의도 (약 1분)

이 서비스를 기획하게 된 이유는, 예전에 AI파인튜닝해서 주간날씨와 지정한위치, 현재위치에 대한 맛집을 추천하는 서비스를 만들려고했었는데, 그 경험을 살려서 이번 프로젝트를 진행하게 되었습니다.

핵심 아이디어는 사용자의 위치를 파악하고, 현재 날씨에 따른 메뉴 추천을 하고, 해당 메뉴를 기준으로 주변 맛집을 탐색할 수 있는 것입니다.

---

## 3. 기술 스택 소개 (약 1분)

이 프로젝트는 주제에 맞게 html, css, javascript를 사용했고, DOM조작과 ajax처리를 위해 jQuery를 사용했습니다.

또한 OPENAPI로는 현재날씨, 5일예보 데이터를 가져오기 위해 OpenWeatherMap을 사용했고, 키워드 기반 검색 및 지도 표시를 위해 Kakao Maps SDK, Kakao Local API를 사용했습니다.

즐겨찾기, 최근 검색어 같은 기능은 사용자데이터를 저장하기 위해 localStorage를 사용했습니다.

---

## 4. 핵심 기능 시연 설명 (약 2분)

### 메인 페이지

사용자가 접속하면 먼저 브라우저의 Geolocation API로 현재 위치를 획득합니다. 그리고 기온, 습도, 풍속, 날씨 상태를 보여주는 날씨 카드가 표시됩니다.

그 아래에는 현재 날씨 조건에 맞는 메뉴 3가지가 추천되고, 각 메뉴 카드에는 "주변에 몇 개의 맛집이 있는지" 실시간 개수도 함께 표시됩니다. 5일 예보 미리보기도 제공합니다.

### 검색 페이지

추천 메뉴를 클릭하면 검색 페이지로 이동합니다. 여기서는 키워드 검색과 내 주변 3km 반경 검색이 가능합니다.

핵심은 **지도와 리스트의 양방향 연동**입니다. 지도의 마커를 클릭하면 왼쪽 리스트에서 해당 카드로 자동 스크롤되고 하이라이트됩니다. 반대로 리스트의 카드에 마우스를 올리면 지도의 해당 마커가 강조됩니다.

카테고리 필터로 한식, 중식, 일식, 양식, 카페를 선택할 수 있고, 거리순, 이름순, 카테고리순 정렬도 지원합니다. 마음에 드는 맛집은 즐겨찾기로 저장할 수 있습니다.

### 즐겨찾기 페이지

저장한 맛집 목록을 확인하고, 개별 삭제 또는 전체 삭제가 가능합니다. 카카오맵 상세 페이지로 바로 이동하는 링크도 제공합니다.

## 5. 핵심 구현 포인트 (약 3~4분)

이 프로젝트에서 기술적으로 신경 쓴 부분들을 설명드리겠습니다.

### (1) `$.when()` 기반 AJAX 병렬 처리

메인 페이지에 진입하면 날씨 API, 예보 API, 역지오코딩 API 총 3개를 호출해야 합니다. 이걸 순차적으로 호출하면 응답 시간이 세 배로 늘어나죠.

그래서 jQuery의 `$.when()`을 사용해서 세 요청을 **동시에** 보내고, 세 개가 모두 완료된 후에 화면을 한 번에 렌더링하도록 했습니다. 이렇게 하면 체감 로딩 시간을 약 1/3 수준으로 줄일 수 있습니다.

또한 역지오코딩 요청은 내부에서 실패해도 `null`로 resolve하도록 래핑해서, Kakao API에 장애가 생겨도 날씨 정보는 정상적으로 표시되도록 설계했습니다.

### (2) Geolocation 2단계 위치 획득

위치 정확도 문제를 해결하기 위해 2단계 전략을 사용했습니다. 1차 시도에서 정확도가 1,200m 이상이면, 자동으로 타임아웃을 늘려서 2차 시도를 수행합니다. 2차도 실패하면 1차 결과를 경고 메시지와 함께 반환하고, 정확도가 1,000m 이상이면 수동 좌표 입력 폼을 표시합니다.

이렇게 하면 GPS 신호가 약한 실내 환경에서도 최대한 정확한 위치를 확보할 수 있습니다.

### (3) 지도-리스트 양방향 연동

Kakao Maps SDK의 마커 이벤트와 jQuery의 DOM 이벤트를 연결해서, 마커 클릭 시 리스트 카드로 스크롤하고 하이라이트하는 기능, 리스트 호버 시 마커를 강조하는 기능을 구현했습니다. `highlightCard()`라는 하나의 함수에서 CSS 클래스 토글, 스크롤, 마커 zIndex 변경, 인포윈도우 열기를 모두 처리하도록 통합했습니다.

### (4) localStorage 안전 저장

즐겨찾기를 localStorage에 저장할 때, 용량 초과 에러인 `QuotaExceededError`가 발생할 수 있습니다. 이때 가장 오래된 항목을 하나씩 제거하면서 재시도하는 로직을 넣어서, 저장 공간이 부족해도 최신 데이터는 보존되도록 했습니다.

### (5) Kakao Maps SDK 동적 로딩

검색 페이지에서만 지도가 필요하기 때문에, SDK를 정적으로 로드하지 않고 `<script>` 태그를 동적으로 생성해서 비동기 로드합니다. `autoload=false` 옵션으로 로딩 시점을 제어하고, jQuery Deferred로 래핑해서 후속 로직을 깔끔하게 체이닝할 수 있게 했습니다. 중복 로딩 방지를 위한 캐싱도 적용했습니다.

## 6. 아키텍처 요약 (약 30초)

전체 구조를 간단히 정리하면, HTML 페이지별로 컨트롤러 역할을 하는 JS 파일이 있고, 공통 유틸, 날씨, 카카오, 즐겨찾기 등 기능별로 모듈을 분리했습니다. 모든 외부 통신은 AJAX로 처리하고, 페이지 간 데이터 전달은 URL 파라미터, 클라이언트 저장은 localStorage를 사용하는 순수 프론트엔드 아키텍처입니다.
