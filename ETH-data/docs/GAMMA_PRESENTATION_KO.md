# When Whales Move
### 시장 이벤트 전후, 큰 자금은 어떻게 움직였을까?

- 이 프로젝트는 **이더리움 온체인 데이터**, **거래소 입출금 흐름**, **가격 데이터**, **뉴스 헤드라인**을 함께 묶어
  시장 이벤트 전후 자금 흐름을 분석하고 시각화한 프로젝트입니다.
- 단순 가격 차트가 아니라, **실제 자금의 행동 데이터**를 중심으로 시장 반응을 읽는 것이 핵심입니다.

**한 줄 소개**  
가격이 아니라 **행동 데이터**로 시장을 읽는 데이터 분석 대시보드

---

# 왜 이 프로젝트를 만들었는가

- 블록체인 시장은 보통 가격 차트 중심으로 설명되지만,
  실제 큰 돈이 어떻게 움직였는지는 잘 보이지 않습니다.
- 뉴스는 많이 보이지만,
  **뉴스가 나왔을 때 실제 자금이 어떻게 반응했는가**는 직관적으로 보기 어렵습니다.
- 그래서 이 프로젝트는
  **시장 이벤트와 실제 자금 이동을 연결해서 보여주는 것**을 목표로 시작했습니다.

### 출발 질문
- 위기 이벤트가 오면 거래소로 자금이 몰리는가?
- 호재 이벤트에서는 거래소 밖으로 자금이 빠져나가는가?
- 한국 이벤트는 글로벌 이벤트와 다른 패턴을 보이는가?

---

# 블록체인 비전공자를 위한 용어 사전

### 고래
- 큰 금액을 움직이는 지갑
- 이 프로젝트에서는 큰 규모의 단일 전송을 고래 행동으로 정의했습니다.

### 거래소 입금
- 외부 지갑에서 거래소 지갑으로 자금이 들어가는 것
- 보통 매도 준비 또는 긴급 대응 신호로 볼 수 있습니다.

### 거래소 출금
- 거래소 지갑에서 외부 지갑으로 자금이 나가는 것
- 보유 전환, 외부 보관, 자금 재배치 신호로 볼 수 있습니다.

### 온체인 데이터
- 블록체인 네트워크 위에 실제로 기록된 거래 데이터
- 뉴스나 의견이 아니라, **실제 행동 기록**이라는 점이 중요합니다.

### 이벤트 윈도우
- 사건 기준 전 7일 ~ 후 7일을 잘라 비교하는 구간

---

# 온체인 데이터는 어디서 가져왔는가

### 주요 데이터 출처
- **Dune Analytics**: 고래 전송 데이터, 거래소 입출금 데이터
- **가격 데이터 CSV**: 이더리움 / 비트코인 일별 가격 데이터
- **이벤트 메타데이터**: 직접 정리한 이벤트 카탈로그
- **뉴스 데이터**: 사건 맥락 설명용 헤드라인 수집
- **공포·탐욕 지수**: 시장 심리 보조 지표

### 왜 Dune을 사용했는가
- 직접 블록체인 인덱서를 구축하지 않아도 됨
- 이미 정리된 이더리움 데이터 위에서 빠르게 분석 가능
- 발표 과제 범위에서 분석 흐름과 시각화에 더 집중할 수 있었음

# 프로젝트 목표와 범위

### 목표
- 위기와 호재 이벤트에서 자금 흐름 차이 비교
- 모든 사건을 **전후 7일 기준**으로 정렬해 반복 패턴 찾기
- 결과를 웹 대시보드로 시각화하기
- 데이터 파이프라인부터 프론트엔드까지 직접 구현하기

### 분석 범위
- 기간: **2017 ~ 2026**
- 체인: **이더리움 메인넷**
- 이벤트 수: **21개**
  - 글로벌 15개
  - 한국 6개

# 시스템 구조

### 전체 구조
**원본 데이터 → 파이썬 파이프라인 → 데이터베이스 → API 서버 → 대시보드**

### 역할 분리
- **파이프라인**: 수집, 정제, 집계, 윈도우 계산
- **데이터베이스**: 분석 기준 저장소
- **API 서버**: 이벤트, 윈도우, 뉴스, 시계열 데이터 조회
- **대시보드**: 사용자가 결과를 탐색하는 화면

### 기술 스택
- **파이썬**: 데이터 정제 / 집계
- **PostgreSQL**: 통합 저장소
- **Fastify + TypeScript**: 읽기 전용 API
- **React + Vite + Recharts**: 시각화 대시보드
- **Docker Compose**: 로컬 재현 환경

# 분석 방법

### 비교 기준
- 모든 사건을 **전 7일 평균 / 사건일 / 후 7일 평균** 기준으로 비교했습니다.
- 서로 다른 시기의 사건도 같은 기준으로 볼 수 있도록 정규화했습니다.

### 왜 ±7일인가?
- 너무 짧으면 노이즈가 크고
- 너무 길면 사건 외 영향이 많이 섞이기 때문입니다.

### 해석 원칙
- 거래소 입금 증가 → 거래소 접근 증가
- 거래소 출금 증가 → 외부 이동 또는 자금 재배치
- 입금과 출금 동시 증가 → 시장 전체 자금 재배치 가능성

# 대시보드 구성

### 1. 메인 타임라인
- 전체 기간의 고래 송금 흐름
- 사건 마커 표시
- 사건 선택 진입점

### 2. 이벤트 상세 분석
- 가설 카드
- 사건 전 7일 평균 = 100 기준의 정규화 차트
- 거래소 순유입 차트
- 가격 / 심리 / 뉴스 보조 맥락 차트

### 3. 이벤트 비교 화면
- 카테고리별 반응 히트맵
- 지역별 반응 히트맵

### 4. 한국 섹터 화면
- 한국 이벤트 6개를 작은 비교 차트 구조로 배치
- 결측 데이터는 명시적으로 표시

# 시각화 설계에서 중요하게 본 점

이 프로젝트에서는 차트를 많이 넣는 것보다,
**한 화면이 한 질문에 답하게 만드는 것**을 더 중요하게 봤습니다.

### 설계 원칙
- 메인 타임라인은 **전체 흐름 인식**에 집중
- 이벤트 상세는 **질문별 패널 분리**
- 비교 화면은 막대그래프 대신 **히트맵**으로 전환
- 한국 이벤트는 카드 목록 대신 **작은 비교 차트 구조**로 비교
- 결측 데이터는 0처럼 보이지 않게 **데이터 없음** 상태로 표시

---

# 프로젝트 진행 중 어려웠던 점과 해결 방법

### 어려웠던 점
- 가격, 거래소 흐름, 뉴스, 이벤트 날짜를 같은 기준으로 맞추는 문제
- 사건을 어떤 기준으로 비교할지 정하는 문제
- 차트가 많아질수록 오히려 읽기 어려워지는 문제
- 결측 데이터를 잘못 해석하지 않게 표현하는 문제

### 해결 방법
- 모든 사건을 **±7일 윈도우** 기준으로 통일
- 분석 로직은 파이프라인에서 미리 계산
- 상세 차트는 질문별 패널로 분리
- 비교 화면은 히트맵으로 단순화
- 결측 데이터는 명시적 상태로 처리

# 주요 분석 결과 요약

### 결과 1
위기 이벤트에서는 거래소 입금이 실제로 크게 증가했습니다.
- 폭락 이벤트 평균 거래소 입금: **+167.4%**

### 결과 2
호재 이벤트에서는 거래소 출금이 거래소 입금보다 더 강하게 반응했습니다.
- 호재 이벤트 평균 거래소 입금: **+28.7%**
- 호재 이벤트 평균 거래소 출금: **+53.6%**

### 결과 3
큰 위기에서는 입금과 출금이 함께 커지며 자금 재배치가 나타났습니다.

### 결과 4
사건 후 7일 이더리움 가격 변화는 시장 해석 방향을 잘 보여줬습니다.
- 폭락 평균: **-33.3%**
- 호재 평균: **+2.8%**

### 결과 5
한국 이벤트는 글로벌 이벤트보다 평균 변화폭이 작았습니다.
- 글로벌 평균 고래 송금량: **+25.3%**
- 한국 평균 고래 송금량: **-11.4%**

# 이 프로젝트를 통해 배운 점

- 데이터 프로젝트에서 가장 중요한 것은 **질문 설계**라는 점
- 좋은 시각화는 많은 차트보다 **읽히는 구조**라는 점
- 파이프라인 / 데이터베이스 / API / 화면을 분리하면 프로젝트 완성도가 높아진다는 점
- 분석 프로젝트도 단순 리포트가 아니라, 사용자가 탐색할 수 있는 제품으로 만들 수 있다는 점

# 결론

이 프로젝트는  
**“시장 이벤트가 발생했을 때, 큰 자금은 실제로 어떻게 움직였는가?”**  
라는 질문에 데이터로 답하려는 시도였습니다.

### 최종 요약
- 위기에서는 거래소 입금과 자금 재배치가 커졌고
- 호재에서는 거래소 출금이 더 강하게 반응했으며
- 한국 이벤트는 글로벌 이벤트보다 평균 변화폭이 작았습니다.

즉, 가격 뒤에는 항상 행동이 있었고,  
이 프로젝트는 그 행동을 읽기 쉽게 정리한 데이터 분석 대시보드입니다.
